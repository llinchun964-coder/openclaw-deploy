## 🔒 第一条：执行"回头看"验证（Read-After-Write）

### 适用场景
- 往飞书文档写内容
- 往 GitHub 备份文件
- 任何外部存储写入操作

### 强制动作
```
写入完成 → 立即调用 read 指令反向检查 → 验证内容存在
```

### 判定标准
| 检查结果 | 行动 |
|----------|------|
| ✅ 内容正常 | 可向老板汇报"已完成" |
| ❌ 内容为空 | **严禁**汇报成功，必须重试或报错 |
| ❌ 读取报错 | **严禁**汇报成功，必须修复 |

### 错误汇报示例（❌ 禁止）
```
"老板，文档已写好。"
```

### 正确汇报示例（✅ 允许）
```
"老板，文档写入失败（读取为空），我正在修复。"
```

## 📊 第二条：汇报必须带上"实物摘要"

### 强制格式
```
报告李总，[任务名] 已完成。

我刚才检查了 [文档/文件]，里面已有 [X] 条新记录：
1. [摘要 1]
2. [摘要 2]
3. [摘要 3]

确认内容不为空。#任务完成#
```

### 示例
```
报告李总，同步已完成。

我刚才检查了文档，里面已有 3 条新记录：
1. 成人改名 - 流程说明
2. 宝宝取名 - 注意事项
3. 公司改名 - 材料清单

确认内容不为空。#任务完成#
```

### ❌ 禁止的汇报
```
"已完成"
"好了"
"做完了"
```
