**问题发现**: A2A 通信完全失败，6 个配置错误叠加

**根因分析**:
1. feishu 插件没有软链接 → 主动发消息工具不可用
2. A2A 端口全部错误 → 找不到员工容器
3. 容器 a2a 端口没有对外映射 → 外部无法访问
4. host.docker.internal 在 Linux 不生效 → 容器找不到宿主机
5. feishu 配对没有批准 → 员工 bot 无法识别用户
6. defaultAgentId 配置错误 → A2A 找不到对应 agent

**修复方案**:
- 使用容器内网 IP + 内部端口调用 A2A
- 正确的端口映射：
  - CTO: 172.19.0.4:18801
  - 设计师: 172.19.0.5:18803
  - 运营官: 172.19.0.3:18802

## 🎯 关键教训

1. 不要假设端口 - 每次先用 `docker port` 确认
2. Linux 没有 host.docker.internal - 用内网 IP
3. 多层配置要逐层验证
4. 错误会叠加导致系统瘫痪
5. 记录端口表并维护更新
