### 3. Google Gemini配置指导
CEO分享了Google Gemini的详细配置指导，要点如下：

#### **核心修复：多Agent的正确配置格式**
- **问题**：OpenClaw 2026.x版本中，agents不允许直接在顶层以自定义ID作为键
- **正确格式**：使用对象映射（Map）格式
- **示例**：
  ```yaml
  agents:
    main:
      name: "首席运营官"
      prompt: "你是COO..."
      workspace: "./workspaces/main"
  ```

#### **飞书多账户与Agent的绑定逻辑**
- **账户配置**：在`channels.feishu.accounts`中定义数组格式账户
- **绑定关系**：在`bindings`中建立映射关系
- **正确格式**：
  ```yaml
  accounts:
    - id: "coo-bot"
      appId: "..."
      appSecret: "..."
  bindings:
    - channel: "feishu"
      accountId: "coo-bot"
      agentId: "main"
  ```

#### **插件安装与Git源问题**
- **memory-lancedb-pro**：需要手动npm install git链接
- **openclaw-engram**：缺少openai对等依赖，需要安装openai

#### **Gateway配置纠正**
- **version**：应该在配置文件最顶层，必须是字符串
- **gateway.auth**：2026版本已弃用authToken，统一使用token
- **正确格式**：
  ```yaml
  gateway:
    host: "0.0.0.0"
    port: 18789
    auth:
      method: "token"
      token: "你的安全令牌"
  ```

## 重要发现

- **Gemini建议**：使用数组格式`[{id: "coo-bot", ...}]`
