# DESIGN QUALITY RUBRIC (XHS Covers)

## Goal
Make cover design quality stable without boss micromanagement.

## Default Output Standard
- Size: `1080x1440` PNG (Xiaohongshu cover standard).
- Deliverables per task:
  - 2 directions per theme (small samples),
  - 1 final per theme after internal selection,
  - editable source file path.
- No "AI generated watermark" look, no inaccessible private links.

## Mandatory Visual Rules
- Brand consistency:
  - fixed brand mark position: bottom-right safe area,
  - fixed typography system (title font + subtitle font),
  - fixed spacing grid and margins.
- Readability:
  - title must be readable on mobile at first glance,
  - strong contrast between text and background,
  - avoid long subtitle lines.
- Commercial quality:
  - avoid generic template look,
  - avoid clutter and random decorative elements,
  - one clear message per cover.

## Scoring (Self-check before submission)
Score each item 0-10. Total 100. Submit only when >= 85.

1. Visual hierarchy clarity (15)
2. Mobile readability (15)
3. Brand consistency (15)
4. Color/contrast quality (10)
5. Composition and spacing (10)
6. Topic-expression match (10)
7. Platform fit (XHS style) (10)
8. Professional/commercial look (10)
9. No artifact/watermark issues (5)

## Hard Rejection Rules
Reject and redo automatically if any of these happens:
- boss cannot open or directly see image in Feishu;
- only text proposal, no actual image;
- obvious AI watermark/artifact remains;
- score < 85;
- missing source file path.

## Feishu Delivery Contract
- Must deliver visible image evidence in Feishu:
  - image visible in doc/group + messageId/screenshot evidence.
- TOS private direct links alone are not acceptable.
