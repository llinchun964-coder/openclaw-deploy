## 📋 各部门方案

### 🔧 CTO 技术部

**1. API 稳定性保障**
- 火山引擎 API Key 轮询（2 个 Key 备用）
- 每日配额监控（剩余 20% 时预警）
- 失败自动重试（3 次）

**2. 审批流程技术实现**
- 飞书 Bitable 自动提交
- 审批状态 webhook 通知
- 发布前自动检查图片

**3. 系统稳定性**
- 容器健康检查（每 5 分钟）
- 自动重启策略
- 每日数据备份

**实施时间**: 3 天
**风险**: 低

---

### 🎨 设计师设计部

**1. 配图风格**
- 主风格：专业 + 古风结合
- 配色：蓝色（专业）+ 金色（高端）
- 字体：思源宋体（标题）+ 黑体（正文）

**2. 提示词模板库**

```
成人改名类：
"专业取名师工作台，古籍，笔墨纸砚，暖光，高清"

宝宝取名类：
"温馨婴儿房，柔和光线，粉色/蓝色调，可爱"

公司取名类：
"现代办公室，商务风格，蓝色调，专业"
```

**3. 出图计划**
- 每天 2 张（1 封面 +1 配图）
- 上午 10 点出图
- 质量检查：CEO 审核

**实施时间**: 立即
**风险**: 低

---

### 📱 运营官运营部

**1. 发布计划**
- 频率：每天 1 篇（20:00 发布）
- 时间：工作日 + 周末
- 备用：周六加发 1 篇（10:00）

**2. 内容比例**
| 类型 | 占比 | 篇数/周 |
|------|------|---------|
| 改名案例 | 40% | 3 篇 |
| 取名知识 | 30% | 2 篇 |
| IP 故事 | 20% | 1 篇 |
| 互动问答 | 10% | 1 篇 |

**3. 审批流程**
1. 运营官起草笔记
2. 提交飞书审批表
3. CEO 审核（24 小时内）
4. 审批通过后发布

**4. 数据追踪**
- 核心指标：点赞/收藏/评论
- 周报：每周日汇总
- 优化：根据数据调整内容

**实施时间**: 立即
**风险**: 中（内容质量波动）

---

### 💼 南南业务部

**1. 咨询流程**
```
客户咨询 → 需求分析 → 报价 → 成交 → 交付 → 反馈
```

**2. 案例收集**
- 每成交 1 单，收集 1 个案例
- 脱敏处理（隐去真实姓名）
- 客户授权确认

**3. 素材提供**
- 每周提供 2 个案例
- 包含：改名前后对比/客户反馈
- 格式：文字 + 截图

**实施时间**: 立即
**风险**: 低
